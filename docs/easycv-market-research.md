# EasyCV 剪贴板管理产品 - 竞品市场调研报告

> 调研日期：2026-02-23  
> 目的：为 EasyCV 产品立项提供市场参考

---

## 一、市场概况

剪贴板管理工具是一个成熟但持续演进的细分市场。用户核心痛点始终未变：**操作系统原生剪贴板只能保存最近一条记录，频繁复制粘贴时极易丢失重要内容。**

### 市场趋势

1. **Apple 入场**：macOS Tahoe（26）已在 Spotlight 中内置剪贴板历史功能，但功能非常基础（详见第二章威胁分析）。这说明 Apple 承认了需求但留下了大量空白。
2. **Windows 同样内置**：Win+V 已提供剪贴板历史，但仅 25 条上限，功能有限（详见第二章威胁分析）。PowerToys Advanced Paste 已在 AI+剪贴板方向探路。
3. **AI 能力尚未渗透**：目前市场上几乎没有剪贴板管理工具集成 AI 能力（如智能分类、内容摘要、语义搜索等），这是潜在的差异化方向。
4. **跨平台需求强烈**：多数用户同时使用 Mac + Windows 甚至 + Linux，但优秀的跨平台方案很少。
5. **订阅疲劳**：用户对小工具的订阅制普遍反感，买断制或免费开源更受欢迎。

## 二、⚠️ 核心风险：系统原生剪贴板历史的威胁分析

> **这是 EasyCV 产品立项必须回答的生死问题：如果 macOS 和 Windows 原生支持足够好，第三方产品将来必死。**

### 2.1 macOS Tahoe（26）原生剪贴板历史

macOS 26 Tahoe（2025.9.15 正式发布）首次在 Spotlight 中内置剪贴板历史，是 Apple 对这一需求的正式回应。

**功能清单：**
- 入口：Spotlight → Clipboard 按钮（Cmd+Space 后 Cmd+4）
- 支持内容：纯文本、图片（显示为 PNG 缩略图）、文件、URL
- 搜索：可在 Spotlight 中搜索剪贴板内容
- 操作：双击粘贴、右键删除/复制/分享、拖拽到其他 App
- 去重：重复复制同一内容会自动移到顶部并删除旧条目
- 密码保护：Apple Passwords App 复制的密码不会进入历史（但第三方密码管理器不保证）
- 保留时长（26.1 更新后）：30 分钟 / 8 小时（默认）/ 7 天 三档可选
- 清除：支持单条删除和一键清空
- 默认关闭，需用户手动开启

**致命缺陷（截至 26.3）：**

| 缺陷 | 严重程度 | 说明 |
|------|---------|------|
| **仅保存纯文本** | 🔴 致命 | 复制富文本时格式全部丢失，只保留文本内容。这意味着无法保存表格、样式等 |
| **最长仅 7 天** | 🔴 致命 | 无法作为长期片段库使用，7 天后自动清除 |
| **无 Pin/收藏** | 🔴 致命 | 无法固定常用片段，所有内容都会过期消失 |
| **无分组/文件夹** | 🟡 重要 | 所有历史平铺展示，无法组织管理 |
| **无编辑** | 🟡 重要 | 复制后无法修改内容再粘贴 |
| **无跨设备同步** | 🔴 致命 | 仅限本机，不同步到 iPhone/iPad（Universal Clipboard 仅支持最新一条） |
| **无团队协作** | 🟡 重要 | 无法与他人共享片段 |
| **文本长度限制** | 🟡 重要 | 约 16,384 字符以上的文本不会被保存 |
| **约 20 条上限** | 🟡 重要 | 历史条目有限，远不如专业工具的无限历史 |
| **安全隐患** | 🔴 致命 | 屏幕录制/共享时剪贴板内容明文暴露（含密码、API Key 等）；第三方密码管理器复制的密码不被过滤 |
| **稳定性问题** | 🟡 重要 | 与第三方剪贴板工具冲突；Apple Watch 解锁方式导致功能失效；Mail App 等无法写入历史 |
| **入口笨拙** | 🟡 重要 | Cmd+Space → Cmd+4 两步操作，MacRumors 用户普遍抱怨"ain't cuttin it" |

**社区评价摘要：**
- MacRumors 用户："barely an MVP"
- 有人希望看到接近 Paste App 的体验，"which feels way more Apple-like"
- Office Watch 评价："a disappointment compared to the Windows version"
- MacMost："If you need long-term storage, pinning, custom organization, or advanced actions, use a third-party manager"

**结论：macOS 原生 = 入门级能力，不构成对专业产品的替代威胁，但会教育市场（用户知道剪贴板历史这个概念了）。**

### 2.2 Windows 11 原生剪贴板历史

Windows 10 1809（2018.10）首次引入，Windows 11 继承并改进 UI。

**功能清单：**
- 入口：Win+V（一步直达，体验好于 macOS）
- 支持内容：文本、HTML、Bitmap 图片（PNG/JPG）
- 条目上限：25 条（硬编码，不可更改）
- 单条大小：4MB 上限
- Pin（固定）：✅ 支持，固定的条目重启不消失
- 云同步：✅ 通过 Microsoft 账户跨 Windows 设备同步
- 纯文本粘贴：✅ Win+V 面板中有"Paste as Text"选项
- 重启清除：未固定的条目重启后全部消失
- 默认关闭，需手动开启

**致命缺陷：**

| 缺陷 | 严重程度 | 说明 |
|------|---------|------|
| **仅 25 条** | 🔴 致命 | 硬编码上限，全天工作很快用完，旧条目自动被挤掉 |
| **4MB 大小限制** | 🟡 重要 | 大截图、大文本块无法保存 |
| **无搜索** | 🔴 致命 | 只能手动滚动列表查找，25 条都翻不方便 |
| **无分组/标签** | 🟡 重要 | 所有条目平铺 |
| **不可调整窗口大小** | 🟡 重要 | 长文本预览困难 |
| **重启清除** | 🟡 重要 | 仅 Pin 的条目存活，其他全丢 |
| **无编辑** | 🟡 重要 | 无法编辑后粘贴 |
| **无 OCR** | 🟡 重要 | 图片中的文字无法提取 |
| **无 App 排除** | 🟡 重要 | 无法设置某些 App 不记录（如密码管理器） |
| **跨平台有限** | 🟡 重要 | 仅 Windows 设备之间同步，可通过 SwiftKey/Phone Link 有限同步到 Android，不支持 Mac/Linux |
| **文件支持弱** | 🟡 重要 | 仅支持 Text/HTML/Bitmap，不支持文件条目 |
| **延迟渲染丢失** | 🟡 重要 | Excel 大表格等延迟渲染的数据可能被静默丢弃 |

### 2.3 PowerToys Advanced Paste（Windows 重要补充）

⚠️ 这是一个需要特别关注的变量。Microsoft PowerToys（开源，GitHub 10万+ stars）的 Advanced Paste 已成为 Windows 剪贴板的"官方增强版"。

**功能（Win+Shift+V 触发）：**
- 格式转换：粘贴为纯文本 / Markdown / JSON / HTML 文件 / PNG 文件
- 本地 OCR：图片中提取文字，完全本地运行
- 音视频转码：转码为 mp3/mp4
- AI 能力（可选）：
  - 支持 OpenAI / Azure OpenAI / Gemini / Mistral 等云端 API
  - 支持 Foundry Local / Ollama 本地模型（无需 API Key，完全离线）
  - 翻译、摘要、语法修正、风格改写、代码生成
  - 自定义 AI Prompt 预设 + 快捷键
- 剪贴板预览：直接显示当前剪贴板内容
- 完全免费开源

**但 Advanced Paste ≠ 剪贴板管理器：**
- ❌ 没有历史记录功能（仅操作当前剪贴板内容）
- ❌ 没有搜索
- ❌ 没有收藏/分组
- ❌ 不是竞品，是"粘贴增强工具"

**意义：** Microsoft 已在 AI + 剪贴板方向探路，但 Advanced Paste 走的是"粘贴时转换"而非"历史管理"的路线。如果 EasyCV 能把两者融合（历史管理 + AI 转换），将比 PowerToys 更完整。

### 2.4 威胁评估矩阵

| 维度 | macOS 原生 | Windows 原生 | 专业产品需求 |
|------|-----------|-------------|-------------|
| 历史条数 | ~20，最长7天 | 25条，重启清除 | 无限，永久 |
| 搜索 | ✅ 基础 | ❌ 无 | 全文/语义/正则 |
| 富文本保留 | ❌ 仅纯文本 | ✅ HTML/Bitmap | ✅ 完整格式 |
| 图片支持 | ✅ 基础 | ✅ 4MB限制 | ✅ 无限制+预览 |
| 文件支持 | ✅ 基础 | ❌ | ✅ |
| Pin/收藏 | ❌ | ✅ | ✅ 分组+标签 |
| 编辑 | ❌ | ❌ | ✅ |
| 云同步 | ❌ | ✅ 仅Windows | ✅ 跨平台E2E加密 |
| OCR | ❌ | ❌（PowerToys有） | ✅ |
| AI | ❌ | ❌（PowerToys有） | 蓝海 |
| 隐私控制 | ⚠️ 有安全隐患 | ⚠️ 基础 | ✅ App排除+加密 |
| 团队协作 | ❌ | ❌ | ✅ |

### 2.5 关键结论

**1. 短期（1-2年）：不构成威胁，反而利好**
- 两个系统的原生方案都是"最小可用产品"级别，和专业工具差距巨大
- 原生功能的最大价值是**教育市场**——让数亿用户第一次知道"剪贴板可以有历史"，为第三方工具创造了更大的潜在用户池
- Maccy（GitHub 13k stars）在 macOS Tahoe 发布后下载量未见明显下降，社区活跃度不减

**2. 中期（3-5年）：需关注但可控**
- Apple 和 Microsoft 都有动力逐步增强原生能力
- macOS 可能添加 Pin、更长保留期、富文本支持
- Windows 可能增加搜索、扩大条目上限
- **但跨平台 + 精致 UI 是系统原生不可能提供的**，这是 EasyCV 的护城河

**3. 安全区域（EasyCV 核心功能中，原生永远不会做的）：**
- 🛡️ **跨平台统一体验**（Apple 不会支持 Windows，Microsoft 不会支持 Mac）
- 🛡️ **视觉化卡片流交互**（系统级工具不会做这么重的 UI）
- 🛡️ **无限历史 + 永久保存**（系统出于隐私/性能考虑必然限制条数和时长）
- 🛡️ **用户自定义分组管理**（系统级工具倾向保持简单）

**4. EasyCV 产品策略建议：**
- ✅ 核心差异化 = **跨平台 + Paste 级体验**，这是系统原生永远不会做的组合
- ✅ 在多源复制（富文本/图片/文件）上直接超越 macOS 原生（只存纯文本）
- ✅ 在搜索上直接超越 Windows 原生（无搜索）
- ✅ 在 Pin+分组上直接超越 macOS 原生（无 Pin）
- ✅ 利用原生功能教育市场后的"升级需求"——"系统自带的不够用？试试 EasyCV"

## 五、EasyCV 产品定位与核心竞争分析

> **产品定位：Paste 的开源 + 跨平台版本**
> 
> 免费开源，类 Paste 交互体验的跨平台剪贴板历史管理工具。
> 核心能力：多源复制（纯文本 / 富文本 / 图片 / 单文件）+ 搜索查找 + 分类钉住

### 5.1 为什么这个定位有机会

目前市面上没有一款产品同时满足"Paste 级体验 + 跨平台"：

| 产品 | 体验好？ | 跨平台？ | 富文本/图片/文件？ | 搜索？ | 分类/Pin？ |
|------|---------|---------|------------------|--------|-----------|
| **Paste** | ⭐⭐⭐ 标杆 | ❌ Apple only | ✅✅✅ | ✅ | Pinboard ✅ |
| **Ditto** | ⭐ UI老旧 | ❌ Windows only | ✅✅ | ✅⭐ | 分组 ✅ |
| **EcoPaste** | ⭐⭐ 尚可 | ✅ 全平台 | ✅✅ | ✅ | 收藏 ✅ |
| **CopyQ** | ⭐ 极客风 | ✅ 全平台 | ✅✅ | ✅ | Tab ✅ |
| **CleanClip** | ⭐⭐ 不错 | ❌ Mac only | ✅✅ | ✅⭐ | 书签 ✅ |
| **Maccy** | ⭐⭐ 极简 | ❌ Mac only | ❌ 仅纯文本 | ✅ | Pin ✅ |
| **macOS 原生** | ⭐ 基础 | ❌ Mac only | ❌ 仅纯文本 | ✅ 基础 | ❌ |
| **Windows 原生** | ⭐ 基础 | ❌ Win only | ⭐ HTML/Bitmap | ❌ | Pin ✅ |

**空白地带 = EasyCV 的机会：**
- Paste 体验最好，但被锁死在 Apple 生态，Windows/Linux 用户无法使用
- EcoPaste/CopyQ 跨平台，但 UI 和交互远不如 Paste，体验粗糙
- 没有一款产品在跨平台的同时提供 Paste 级别的视觉化卡片流体验

### 5.2 EasyCV 核心功能拆解 vs 竞品对标

**① 多源复制支持**

| 复制类型 | EasyCV 目标 | 对标参考 | 需要注意的坑 |
|----------|-----------|---------|-------------|
| **纯文本** | ✅ 必须 | 所有竞品都支持 | 基础能力，不是差异点 |
| **富文本** | ✅ 保留完整格式 | Paste（最好）、Ditto（RTF/HTML） | macOS 原生只存纯文本——这是 EasyCV 的直接优势点 |
| **图片** | ✅ 缩略图预览 + 原图粘贴 | Paste（富预览）、EcoPaste | 注意大图的存储和性能；需要设置单张大小上限 |
| **单文件** | ✅ 记录文件路径 + 类型图标 | Paste、Ditto、EcoPaste | 文件可能被移动/删除——需处理失效路径提示 |

**关键差异点：** macOS 原生和 Maccy 都不支持富文本/图片/文件。EasyCV 在这三项上的完整支持，就是对系统原生的直接超越。

**② 搜索查找**

| 搜索能力 | EasyCV 目标 | 竞品基准线 | 超越点 |
|----------|-----------|-----------|--------|
| 文本内容搜索 | ✅ 必须 | 所有竞品都支持 | 基础能力 |
| 模糊搜索 | ✅ 推荐 | CleanClip（拼音搜索） | 中文拼音首字母搜索是加分项 |
| 按类型过滤 | ✅ 推荐 | 少数竞品支持 | 快速筛选"只看图片"/"只看文本" |
| 按来源 App 过滤 | 🔜 可考虑 | CleanClip 独有 | 低优先级但有价值 |
| 按时间范围搜索 | ✅ 推荐 | 少数竞品支持 | "今天"/"本周"/"上月"快筛 |

**注意：** Windows 原生完全没有搜索——这是 EasyCV 在 Windows 上最直接的卖点之一。

**③ 分类钉住**

| 组织能力 | EasyCV 目标 | 对标竞品 | 说明 |
|----------|-----------|---------|------|
| Pin/收藏 | ✅ 必须 | Paste（Pinboard）、Windows 原生（Pin） | 固定常用片段，不会过期消失 |
| 分组/文件夹 | ✅ 推荐 | Paste（Pinboard 可分多板）、Ditto（分组）、CopyQ（Tab） | 让用户自己组织内容，如"工作"/"代码"/"地址" |
| 智能分组 | 🔜 可考虑 | EcoPaste（自动按类型分组） | 自动按内容类型（文本/图片/链接/文件）分组展示 |
| 标签 | 🔜 低优先级 | 无竞品做 | V2 再考虑 |

**关键：** macOS 原生完全不支持 Pin——用户复制的重要内容 7 天后就消失。EasyCV 的 Pin + 分组是核心留存功能。

### 5.3 交互体验对标 Paste

Paste 的体验标杆在于：

1. **视觉化卡片流**：每条记录以卡片形式展示，富文本/图片有直觉的预览，不是纯文本列表
2. **快捷键直达**：一个快捷键呼出，不用像 macOS 原生那样 Cmd+Space → Cmd+4 两步
3. **键盘优先**：方向键浏览、回车粘贴、输入即搜索
4. **上下文保留**：显示来源 App 图标、复制时间
5. **Paste Stack**：依次复制多条 → 依次粘贴（批量操作场景）

**EasyCV 至少需要做到 1-4 才能对标，第 5 项 Paste Stack 可以作为差异化的加分项。**

### 5.4 跨平台是最大且唯一的差异化

说直白一点：如果 EasyCV 只做 Mac，没有理由选它而不是 Paste/CleanClip/Maccy。如果 EasyCV 只做 Windows，没有理由选它而不是 Ditto。

**跨平台（Mac + Windows，Linux 可选）是立项的核心前提。** 具体价值：

- 大量开发者/设计师同时使用 Mac + Windows
- 企业场景中不同部门用不同系统
- 目前跨平台+体验好的方案 = 空白
- 系统原生方案永远不会跨平台（Apple 不会支持 Windows，反之亦然）——这是天然护城河

### 5.5 相对原生的核心优势总结

| 原生做不到 / 做得差 | EasyCV 做到 |
|-------------------|-----------|
| macOS 只存纯文本 | ✅ 富文本 + 图片 + 文件 |
| macOS 最长 7 天 / Windows 25 条 | ✅ 无限历史，永久保存 |
| macOS 无 Pin / Windows Pin 但无分组 | ✅ Pin + 分组 |
| Windows 无搜索 | ✅ 全文搜索 + 过滤 |
| 两个系统都不跨平台 | ✅ Mac + Windows 统一体验 |
| 入口笨拙（macOS Cmd+Space+4） | ✅ 单快捷键直达 |
| 安全隐患（密码明文、屏幕共享暴露） | ✅ App 排除规则 |
| UI 基础（列表/小窗） | ✅ Paste 级视觉化卡片 |

## 八、总结与行动建议

### 产品一句话定义

> **EasyCV = Paste 的开源跨平台替代**
> 
> 免费开源。在 Mac 和 Windows 上提供统一的、Paste 级视觉化卡片体验的剪贴板历史管理工具，支持纯文本/富文本/图片/文件的完整复制记录、全文搜索和分类钉住。

### MVP 功能清单（V1.0）

| 优先级 | 功能 | 原因 |
|--------|------|------|
| P0 | 剪贴板监听 + 无限历史存储 | 产品存在的基础 |
| P0 | 多源支持：纯文本、富文本、图片、单文件 | 直接超越 macOS 原生（只存纯文本）和 Maccy |
| P0 | 视觉化卡片预览（对标 Paste） | 核心体验差异化，不能做成文本列表 |
| P0 | 全局快捷键一键呼出 | 效率工具的基本要求 |
| P0 | 搜索（全文 + 按类型过滤） | 直接超越 Windows 原生（无搜索） |
| P0 | Pin 收藏 + 用户自定义分组 | 直接超越 macOS 原生（无 Pin），核心留存功能 |
| P0 | 跨平台：Mac + Windows | 立项核心前提，唯一且最大的差异化 |
| P1 | 来源 App 图标 + 复制时间显示 | 上下文信息，帮助快速定位 |
| P1 | 纯文本粘贴选项 | 常见需求，所有竞品都支持 |

### 不做的事（至少 V1 不做）

- ❌ AI 能力（语义搜索、智能摘要等）——保持工具纯粹
- ❌ 云同步 / 跨设备同步——复杂度高，V1 专注本地
- ❌ 团队协作 / 共享片段库
- ❌ OCR / 格式转换 / 代码高亮
- ❌ 移动端（iOS/Android）
- ❌ 内容编辑

```
用户选择 EasyCV 的理由：

Mac 用户：  "系统原生只存纯文本，Paste 要 $30/年或 $90 买断"
            → EasyCV 免费开源，富文本+图片+文件，视觉化体验，且换 Windows 也能用

Win 用户：  "Win+V 只有 25 条还没搜索，Ditto 功能够但 UI 太丑"
            → EasyCV 无限历史 + 搜索 + 现代 UI，且换 Mac 也能用

双系统用户："Mac 上用 Paste，Windows 上用 Ditto，体验割裂，数据不通"
            → EasyCV 一套工具两个平台，体验统一，免费开源
```
